# 缺失的第一个正数（First Missing Positive）

## 题目描述
给你一个未排序的整数数组 `nums`，请你找出其中**没有出现的最小正整数**。

要求：时间复杂度 \(O(n)\)，且只使用**常数级额外空间**（原地操作数组）。

## 关键观察
设数组长度为 `n`，答案一定在区间 \([1, n+1]\) 内：
- 如果 `1..n` 都出现了，那么最小缺失正数只能是 `n+1`
- 否则缺失的一定落在 `1..n` 之间

例子：
- `nums = [1,2,3]`，`n=3`，答案 `4`
- `nums = [3,4,-1,1]`，`n=4`，答案 `2`

## 核心技巧（原地哈希 / 置换）
把数组本身当作“哈希表”：让 **下标 `i` 的位置存放值 `i+1`**。

也就是说，数字 `x`（若 `1 <= x <= n`）应该放到下标 `x-1` 的位置。
当我们尽量把所有可放置的数字都放回“正确位置”后：
- 再扫一遍数组，找到第一个满足 `nums[i] != i+1` 的位置，则答案为 `i+1`
- 若全部满足，则答案为 `n+1`

## 具体步骤
1. 遍历数组下标 `i`，当 `nums[i]` 是可放置的值（`1..n`）且它不在正确位置上时，就把它交换到目标位置 `nums[i]-1`。
2. 交换后 `nums[i]` 会变成新的值，继续尝试放置，直到当前位置无法再放（超范围 / 已在正确位置 / 目标位置已有相同值）。
3. 第二次遍历数组，返回第一个 `nums[i] != i+1` 的 `i+1`；若都匹配，返回 `n+1`。

## 为什么不会死循环
内层循环条件包含 `nums[nums[i]-1] != nums[i]`：
- 当数组里有重复值（例如两个 `2`）时，目标位置已经是 `2`，就不会再交换，避免在两处来回换导致死循环。

## Go 实现

```go
func firstMissingPositive(nums []int) int {
    n := len(nums)
    for i := 0; i < n; i++ {
        // 将 nums[i] 放到它应该在的位置：index = nums[i] - 1
        for nums[i] >= 1 && nums[i] <= n && nums[nums[i]-1] != nums[i] {
            idx := nums[i] - 1
            nums[idx], nums[i] = nums[i], nums[idx]
        }
    }
    for i := 0; i < n; i++ {
        if nums[i] != i + 1 {
            return i + 1
        }
    }
    return n + 1
}

```

## 复杂度
- **时间复杂度**：\(O(n)\)。每个元素最多被交换到位一次（总交换次数线性级别）。
- **空间复杂度**：\(O(1)\)。只在原数组上交换，使用常数额外变量。
